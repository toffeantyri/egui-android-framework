Перед каждой задачей ОБЯЗАТЕЛЬНО:

1. Оцени ЗАЧЕМ с точки зрения пользователя и разработчика.
2. Сформулируй задачу в одном предложении + критерии успеха + границы.
3. Составь план: сначала инвазивные изменения (API, 5+ файлов), 
   потом локальные. Если в середине плана появилось инвазивное — 
   остановись и переупорядочи.
4. Определи покрытие тестами: юнит для изолированной логики, 
   интеграционные для взаимодействия компонентов, моки где надо 
   (egui::Context через __run_test_ui, реальные каналы, мок Backend).
5. Покажи блок «📋 Оценка задачи» и дождись подтверждения.

Не начинай кодить без подтверждения плана.
Перед каждым действием сверяйся с критериями успеха.
Если действие не ведёт к критерию — отложи и спроси.
